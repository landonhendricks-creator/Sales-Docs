Stage 2 → 3
Business Objectives
  • Add a formal Contact Center solution that sits ON TOP of Microsoft Teams — preserving existing Teams environment while adding enterprise CC capabilities
  • Achieve a true partnership with a vendor who provides dedicated US-based account management and ongoing support (not just a transactional vendor relationship)
  • Gain deep real-time visibility into contact center performance — dashboards, SLAs, queue health, and agent activity without switching applications
  • Implement formal disposition codes, skills-based routing, and structured wrap-up workflows that current Teams calling plan cannot support
  • Replace manual call auditing with AI-driven QMSA — speech analytics, sentiment mapping, keyword detection, and automated evaluation templates
  • Scale CC infrastructure to support ~65 SOC agents with concurrent licensing as business grows
Challenges & Pain Points
  • Teams calling plan lacks contact center depth — no disposition codes, no skills-based routing, no QMSA, no formal agent/supervisor workspace
  • No real-time supervisor alerting for queue issues — if calls pile up or agents go dark, leadership has no proactive notification path. Currently reactive.
  • Manual call auditing is time-consuming and incomplete — Erica's team listens to recordings manually; no AI, no keyword categorization, no sentiment mapping
  • Agent call volume balancing is ad hoc — no native way to limit calls per hour for new hires vs. senior agents; creates risk of overloading or underloading staff
  • Call Cabinet (Smarsh) may be redundant — compliance recording currently done via Call Cabinet; 8x8 has native recording capability; overlap not yet resolved
  • Fragmented reporting — no unified view of CC + UC data; pulling reports from separate systems creates overhead and delays leadership insight
Strategies
  • Deploy 8x8 CC (X6/X7/X8) on top of existing Teams environment — direct routing as a service; CC agents remain in Teams tenant but gain full 8x8 enterprise CC capabilities
  • Implement skills-based routing across 2-3 existing queues (expandable) — route calls to highest-skilled available agent; overflow to secondary skill tiers
  • Enable QMSA — replace manual audits with AI-driven sentiment analysis, keyword/phrase categorization, agent action item extraction, and evaluation templates
  • Scope Teams webhook/API integration for queue overflow alerts — ping Leadership channel in Teams when queue thresholds are breached (action item for Landon)
  • Concurrent licensing model — only pay for max simultaneous agents logged in; cost-efficient for shift-based SOC operation
  • Evaluate Call Cabinet overlap — determine if 8x8 native recording can replace or supplement Call Cabinet/Smarsh for compliance recording

Future State / Positive Business Outcomes
  • Agents work in a unified environment — Teams for internal collaboration, 8x8 CC workspace for member interactions; one platform, minimal context switching
  • Supervisors see live queue health at a glance — customizable dashboards with SLA, abandons, agent status; get proactively pinged in Teams when thresholds are breached
  • QMSA eliminates manual audits — AI categorizes and surfaces the calls that need review; Erica's team coaches with data instead of guesswork
  • Formal disposition codes provide structured call outcome data — enables reporting by call type, identifies trends, feeds scheduled reports to leadership
  • Skills-based routing delivers callers to the right agent faster — new hires protected from overload; senior agents used for high-priority queues
  • Unified reporting — CC and UC data in one platform; scheduled reports delivered to leadership inbox weekly without manual effort
Required Capabilities
CC on Top of Teams (Direct Routing as a Service)
8x8 CC overlays existing Teams environment. CC agents remain in Teams tenant. UCaaS NOT needed — Joseph explicitly stated no interest in 8x8 UC replacing Teams calling plan. Contact center sits on top.
✓ Confirmed — Teams-Native CC
Skills-Based Routing
Assign skill levels to agents within each queue. High-skill agents receive calls first; overflow to lower-skill tiers. Longest-idle routing within skill tier. Currently 2–3 queues; no limit on queue count.
✓ Confirmed 8x8
Disposition Codes
Currently unavailable in Teams calling plan — Erica and Joseph confirmed they want them. 8x8 CC supports disposition codes natively. Available in X6/X7/X8 formal CC tiers (NOT in Engage/lightweight model).
✓ Confirmed — Requires X6+ CC License
QMSA — Speech Analytics & AI Call Review
Replaces manual call audits. AI categorizes calls by topic (agent confrontation, empathy, compliance phrases). Sentiment mapping, full transcript, agent action item extraction, keyword phrase detection. Evaluation templates (yes/no, sliding scale, auto-fail).
✓ Confirmed 8x8
Real-Time Dashboards + Customizable Reporting
Build custom dashboards with any CC metric (SLA %, abandons, agent status, queue depth). Shareable via URL. Scheduled reports emailed to leadership/distribution lists. Erica confirmed this is critical. Journey map also well-received by Joseph.
✓ Confirmed 8x8
Barge / Monitor / Whisper
Supervisors can silently monitor agents (agent and caller unaware), whisper coaching to agents mid-call, or barge into call when needed. Available in supervisor workspace. PJ requested Tom demo this — well received.
✓ Confirmed 8x8
Queue Alert / Webhook to Teams Channel
Joseph's request: when a queue backs up or thresholds are breached, ping a Leadership Alerts Teams channel via webhook — more actionable than email. Not out of box. Landon action item to scope feasibility via 8x8 API or pro services.
⚠ Not Native — Landon Scoping
Call Volume / Per-Agent Hourly Limits
Erica requested ability to limit calls per hour per agent (protect new hires from overload). Not a native 8x8 inbound CC feature. Workarounds: skill-level manipulation, manual supervisor status changes, overflow queue design. Erica noted Five9 has this natively — GAP.
⚠ GAP — No Native Limit; Workarounds Available
Compliance Recording (Call Cabinet / Smarsh Review)
Currently using Call Cabinet (Smarsh) for compliance recording of all inbound + outbound calls. 8x8 has native recording. Joseph asked if 8x8 can replace Call Cabinet. Overlap to be evaluated in scoping — may reduce cost or simplify stack.
⚠ Overlap TBD — Scoping Required
Queue Recording by Type (CC Only — Not Internal Teams)
Erica confirmed: record CC queue calls and external outbound only — NOT internal Teams-to-Teams calls. 8x8 recording scoped to specific queues, configurable at 100% / partial. Internal Teams calls remain unrecorded.

Auto Wrap-Up / After Call Work Timer
Erica asked if wrap-up status auto-returns agent to available after a set time. Confirmed: configurable after-call work (ACW) timer; agent auto-returns to available after timer expires. Supervisor can also manually override agent status.
