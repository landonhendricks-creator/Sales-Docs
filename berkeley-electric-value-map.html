Account: Berkeley Electric Cooperative — Moncks Corner, SC (Berkeley, Charleston, Dorchester counties) | AE: Stacie Marquardt | SE: Tom Rupp | Champion: Tony Shepherd (Dir. IT Ops) | EB: Amy Langdon (VP IT) ✓ In room | Partner: Mike Hancock / Inspire Communication | Demo: Apr 2, 2026
Stage 2
Business Objectives
  • Modernize off aging Mitel on-prem platform with a unified cloud UCaaS + CCaaS solution across 300 users and 25 CC agents
  • Ensure mission-critical uptime and communication resilience during storm and hurricane events when agents log in across all district offices
  • Improve contact center quality management through AI-driven sentiment analysis, real-time supervisor alerting, and automated QA scoring
  • Consolidate disparate tools (Mitel, Zoho chat, CIS IVR) under a simpler platform without displacing core CIS/IVR workflows
  • Build a Snowflake data warehouse fed by 8x8 API data to enable unified reporting and custom dashboards for leadership
  • Simplify IT administration — AD/Entra sync, profile-based provisioning, SSO, and hot desking to reduce day-to-day management overhead
Challenges & Pain Points
  • Mitel on-prem is aging — limited roadmap, on-prem risk, and no path to the cloud without replacement. Tony has prior experience with Fuse and RingCentral — actively evaluating options
  • Mitel reporting is poor ("not great") — contact center reporting inadequate for supervisors managing 25 agents across multiple queues
  • No real-time sentiment or supervisor alerting — if a call goes negative, supervisors have no automatic notification to intervene
  • Zoho chat is siloed from voice — chat and voice are on separate platforms, creating a fragmented omnichannel experience
  • Mitel softphone add-ons cost extra per user — every feature requires an additional license; 8x8 X Series seat is all-inclusive
  • Storm/hurricane scenarios require agents at district offices — current system lacks the flexibility to seamlessly distribute contact center agents during mass outage events
  • No Snowflake integration today — call and agent data can't be correlated with other operational data for board-level insight
Strategies
  • Replace Mitel on-prem with 8x8 X Series — all-inclusive seat licensing covering UCaaS, CC, DID, mobile app, desktop client, and Teams integration
  • Deploy 8x8 QMSA (Quality Management & Speech Analytics) for sentiment analysis, topic categorization, auto-scoring, and real-time agent assist supervisor alerts
  • Integrate 8x8 with Snowflake via open APIs — pull call, queue, and agent data into Berkeley's data warehouse for custom dashboards
  • Teams cross-launch integration (E3, no Phone license) — embed 8x8 voice widget inside Teams so users don't switch apps
  • Retain CIS/NIS IVR for balance, outage, and payment self-service — 8x8 routes overflow to live agents; no board-approved IVR replacement needed
  • Entra ID (Active Directory) sync + field mapping + SSO — automate provisioning; disable AD user = disable 8x8 phone automatically
  • Reuse Mitel 6930/6940 SIP phones — 5000 series retired as paperweights; 6900 series reused to reduce hardware refresh cost
